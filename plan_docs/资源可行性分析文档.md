## 数据资源

### 1. 穿搭与审美数据

**数据来源与获取渠道**

| 数据类型      | 主要来源             | 获取方式                | 难度 | 合规性      |
| ------------- | -------------------- | ----------------------- | ---- | ----------- |
| 穿搭图片库    | 小红书、抖音、微博   | 公开API或合作协议       | 中等 | ✅ 需授权   |
| 服装商品数据  | 淘宝、京东、拼多多   | 联盟API（淘宝开放平台） | 低   | ✅ 官方支持 |
| 风格/搭配数据 | 时尚媒体、期刊社     | 内容合作或爬虫采集      | 中等 | ⚠️ 需协议   |
| 电商SKU数据   | 得物、识货、iFashion | 官方开放API或爬虫       | 中等 | ⚠️ 需评估   |

**获取方案**：

- **第一阶段（MVP）**: 利用公开Kaggle数据集 + Hugging Face 模型库，快速验证技术
- **第二阶段（商用）**: 与淘宝联盟、京东联盟官方合作，获得授权商品库 API
- **第三阶段（规模化）**: 与MCN机构、设计师品牌建立内容合作协议，获取高质量穿搭数据

**成本**: 初期零成本（开源数据），商用阶段约 **10-50万/年** 数据合作费

### 2. 用户身体数据

**数据采集与存储方案**

**采集方式**:

- 用户上传正面/侧面全身照（可选，非强制）
- 用户输入简单尺码信息（身高、体重、穿衣尺码）
- 通过人体姿态估计模型自动计算关键尺寸

**合规存储方案**:

```
┌─────────────────────────────────────────┐
│         用户身体数据安全框架              │
├─────────────────────────────────────────┤
│ 1. 本地存储优先                          │
│    - 关键身体数据存储在用户设备本地      │
│    - 加密算法: AES-256                   │
│                                          │
│ 2. 隐私保护                              │
│    - 上传数据前自动去标识化处理          │
│    - 剥离与身份直接关联的信息            │
│    - 遵循《个人信息保护法》              │
│                                          │
│ 3. 用户权益                              │
│    - 明确的数据收集、使用、删除权        │
│    - 提供数据导出功能                    │
│    - 随时可撤销授权                      │
└─────────────────────────────────────────┘
```

**合规风险与应对**:

- **法律咨询**: 需聘请数据合规律师 **5-10万** 咨询费
- **认证申请**: 信息安全等级保护（等保二级）**5-15万** 申请和审核费
- **持续合规**: 建立法务团队或外包合规监察 **10-20万/年**
